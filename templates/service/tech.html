사용 기술 이름 및 설명

사용 이유 그리고 장점을 간단히 기술합니다.

ELK

SQL 문법을 통한 경직된 검색이 아닌 google 검색과 같은 유연한 검색기능을 사용하기 위해 ELK stack을 사용하였습니다.
* 추가적으로 logstash 및 filebeat를 통해 로그 수집 기능 역시 구현했습니다.
* a1과 A1을 다른 문자가 아닌 같은 문자로 인식해 검색하는 유연함을 보여줍니다.

  • Learn more

Django

웹페이지 구현을 위해 Django Framework를 사용했습니다.
* Elasticsearch Query를 django의 view에서 python 문법으로 작성 후 바로 Elasticsearch와 통신이 가능하다는 장점이 있습니다
* 위의 통신과 ajax를 결합해 반응형 웹사이트 구축이 용이하다는 장점 역시 있습니다

  • Learn more

GCP

호스팅, 분석, ELK 설치 위한 클라우드 컴퓨팅 서비스로 GCP를 이용하였습니다.
* AWS 대신 사용한 이유는 3개월간 고성능의 인스턴스를 사실상 무료로 이용 가능하기 때문입니다.

  • Learn more

Mariadb

로그인 기능을 구현하기 위해 관계형 db인 maria db를 사용하였습니다

  • Learn more

Hadoop

업로드된 이미지 파일을 관리하기 위해 hadoop을 사용했습니다
* 사용자가 늘어날 경우 업로드된 이미지의 효율적인 관리를 보장합니다.

  • Learn more

Airflow

일련의 프로세스를 자동화 하기 위해 Airflow를 사용하였습니다
* 추후 코드의 유지보수를 위한 합리적인 방안입니다
